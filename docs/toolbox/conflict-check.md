# Mod 冲突检测

“Mod 冲突检测”用于检查多个 VPK 是否包含相同路径的资源文件。

## 作用

当两个 Mod 修改同一个模型、材质、脚本或音频时，后加载的文件可能覆盖前面的文件。冲突检测可以列出这些覆盖关系。

## 使用步骤

1. 打开“工具箱”。
2. 点击“Mod 冲突检测”。
3. 等待扫描完成。
4. 查看冲突文件组和涉及的 Mod。

## 结果怎么看

结果通常会按严重程度显示：

- 严重：可能直接影响游戏内容或导致明显覆盖。
- 警告：可能有影响，需要结合实际 Mod 判断。
- 信息：存在重复文件，但不一定有问题。

## 如何处理

常见处理方式：

- 禁用其中一个 Mod。
- 调整加载顺序。
- 给 Mod 添加标签，避免以后同时启用。
- 如果只是相同的说明文件或预览资源，可以忽略。

## 注意事项

冲突检测只能发现“文件路径相同”的覆盖关系，不能保证判断所有脚本逻辑冲突或运行时问题。
